GIT
    * es un controlador de versiones VCS (Version Control System)
    * Open Source
    * software que nos facilita el tener un control sobre un  proyecto, teniendo un listado de los cambios que se hacen sobre este, facilitando que varios desarroladores trabajen a  la vez sobre un proyecto.
    * Revertir cambios  ya sea parcial o total
    * con git puedes trabajar con repositorios locales o remotos
    

Que hacer?
    
    * seguir los cambios de codigo
    * snapShots (versiones de los codigos)
    * ayudar a decidir cuando hacer los cambios

  
Existen 3 estados en GIT
    
    working directory:
        Donde yo trabajo con todos mis archivos
    
    Staging area: 
        Donse se agregan todos los archivos que se van a guardar 

    Repository:
        Cuando ya esta decidido todo los cambios, se pasaran a guardar en el repositorio




COMANDOS BASICOS
        git init
            para crear un proyecto nuevo, decirle que se utilizara GIT en el proyecto

        git add <FILE>
            pasar los archivos de working directory al staging area

        git status
            saber el estado de los archivos ( si estan en el workign directory, staging area,... )

        git commit
            pasar del staging area al repository, crear un SNAPSHOT (una version del codigo)

        git push
            subir el cambio a un repositorio remoto (lugar donde los demas desarrolladores podan acceder a los cambios realizados)

        git pull
            cuando se esta trabajando con multiples desarrolladores, te traes los cambios que han echo los demas

        git clone
            hacer una copia del servidor central donde esta codigo a mi computadora

PRIMEROS PASOS
    
    en consola GIT bash, nos ponemos en el path del proyecto que quremos trabajar con GIT y corremos el comando [ git init  ].
    esto creara folder oculto [.git]

    con [ git status] podremos ver los archivos que tenemos en que trabajamos. los archivos que aparezcan en rojo, nos quiere decir que no estan en mi staging area

    usamos el comando [ git add myScript.js ] ahora este archivo cuando corra el git status me aparecera en verde y los demas en rojo.

    una vez agregados todos los archivos pasaremos hacer el primer commit utilizando [git commit]
    antes, tenemos que configurar unas variables globales para agrgar al commit informaion de quien hizo el cambio:
    [git config --global user.name "myNAme"]
    [git config --global user.email "ulises@x.com"]
    despues de configurar ahora si corremos [git commit]

    
    ahora la consola cambia, es un editor de codigo en la terminal [llamado VIM]
    en caso que no se pueda escribir, typear la letra 'i' para poder escribir 
    cuando escribamos nombre del commit, ahora escribimos [:wq]
    en este punto ya se debio de crear el commit

    [git log] nos muestra la informacion de el proyecto,, respecto a los commits y quien los hizo
